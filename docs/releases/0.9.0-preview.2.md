# BotFramework 0.9.0-preview.2

`0.9.0-preview.2` is a corrected framework preview release. It keeps the
public API and runtime changes from `0.9.0-preview.1` and fixes the NuGet
template package identity so the complete package-only surface can be
published.

## Packaging correction

The package id `BotFramework.Templates` already exists on NuGet under another
owner. The framework template is therefore published as
`BotFramework.GameTemplates`; its CLI short name remains
`botframework-game`:

```bash
dotnet new install BotFramework.GameTemplates::0.9.0-preview.2
dotnet new botframework-game -n CoinFlip --module-id coin-flip
```

The existing preview package ids and the template's generated project surface
remain unchanged. Consumers should use the new package id for the template.

## Published package set

| Package | Purpose |
|---|---|
| `BotFramework.Contracts` | Tenant, transport, error, pagination and limiter contracts |
| `BotFramework.Sdk` | Module, domain, decision/effect and scheduling abstractions |
| `BotFramework.Testing` | Test fakes and in-memory fixtures |
| `BotFramework.Rest` | Tenant-aware REST middleware and module route contract |
| `BotFramework.Telegram.Abstractions` | Telegram update and tenant-resolution contracts |
| `BotFramework.Discord.Abstractions` | Discord tenant-resolution contracts |
| `BotFramework.Client` | Typed REST client and generated transport |
| `BotFramework.Scheduling.Abstractions` | Transport-neutral scheduled command contracts |
| `BotFramework.GameTemplates` | `dotnet new botframework-game` project template |

`BotFramework.Host`, `BotFramework.Telegram`, `BotFramework.Discord`,
`BotFramework.Scheduling.Quartz` and `BotFramework.Rendering` remain runtime
composition projects. Durable workflows are still consumed by repository
composition roots and source-referenced examples rather than published as a
standalone Host package.

## Validation

The release workflow runs the repository build and tests, public API checks,
package consumer smoke tests, template consumer smoke tests and NuGet package
validation before publishing the nine packages and creating the GitHub
Release.
